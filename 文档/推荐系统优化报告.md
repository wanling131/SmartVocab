# SmartVocab 掙荐系统优化报告

## 概述

本次优化对 SmartVocab 的推荐系统进行了全面升级，引入了多项先进技术。

## 优化内容
### 1. 协同过滤模块 (CollaborativeFiltering)
- **基于用户的协同过滤**: 通过余弦相似度找到相似用户，推荐他们学习过但目标用户未学习的单词
- **基于物品的协同过滤**: 使用 Jaccard 相似度计算单词之间的共现关系
- **用户-物品矩阵缓存**: 优化重复查询性能

### 2. 动态权重调整器 (DynamicWeightAdjuster)
- **个性化权重**: 根据用户反馈动态调整各算法权重
- **UCB 算法**: 使用 Upper Confidence Bound 算法进行探索与利用平衡
- **探索率衰减**: epsilon-greedy 策略，探索率随学习轮次衰减
- **算法效果追踪**: 记录每个算法的成功率和掌握度提升
### 3. 多样性控制器 (DiversityController)
- **MMR 算法**: Maximal Marginal Relevance 算法确保推荐多样性
- **词性限制**: 限制同一词性单词的比例
- **难度分布**: 确保推荐结果的难度跨度合理
### 4. 冷启动处理器 (ColdStartHandler)
- **热门词策略**: 推荐被最多用户学习过的词
- **基础难度词**: 为新用户推荐简单词汇
- **冷启动检测**: 自动识别冷启动用户
### 5. 实时个性化器 (RealtimePersonalizer)
- **会话跟踪**: 跟踪用户当前学习会话表现
- **动态难度调整**: 根据正确率和响应时间调整推荐难度
- **疲劳检测**: 识别用户疲劳状态
- **参与度评估**: 评估用户学习参与度
### 6. 深度学习模型增强
- **注意力机制**: 增加交叉注意力机制提升模型表达能力
- **LayerNorm**: 使用 LayerNorm 替代 BatchNorm 支持小批量推理
- **增强特征**: 单词特征从 20 维扩展到 25 维，- **用户特征从 15 维扩展到 25 维**:
  - **特征工程**: 添加学习模式、多样性偏好、实时表现等特征

## API 增强
### 新增/修复方法:
- `record_feedback()`: 记录用户反馈用于权重调整（已修复：现在在答题流程中被正确调用）
- `end_learning_session()`: 结束学习会话并返回摘要

### 性能优化:
- **预加载共享数据**: `_get_enhanced_mixed_recommendations()` 入口一次查询 `list_all()` + `get_by_user()`，通过参数共享给 4 个子算法，消除重复 DB 查询（单次推荐查询数从 12+ 降至 2）
- **N+1 查询修复**: 遗忘曲线 `_enrich_with_word_details()` 和学习记录 `get_user_learning_records()` 改用 `get_by_ids()` 批量查询，含 fallback 降级
- **冷启动策略增强**: `get_cold_start_recommendations()` 现在读取等级测试结果（`evaluation_results`），根据 `assessed_level` 动态调整难度范围，不再固定 [1, 2]

### 自检修复:
- **循环导入修复**: `LEVEL_TO_DIFFICULTY` 改为方法内懒加载，避免推荐增强模块全部失效
- **response_time 处理**: 前端不传时正确使用估算值（正确 10 秒/错误 20 秒），不再使用 0
- **新单词反馈**: `mastery_after` 正确使用创建时设定的掌握度值

### 协同过滤推荐:
- `_get_collaborative_recommendations()`: 基于用户的协同过滤
- `get_item_based_recommendations()`: 基于物品的协同过滤

## 使用示例
```python
# 获取推荐（自动使用增强功能）
engine = RecommendationEngine()
recommendations = engine.get_recommendations(user_id=1, limit=10)

# 记录反馈（用于动态权重调整）
engine.record_feedback(
    user_id=1,
    word_id=100,
    algorithm_type='deep_learning',
    mastery_before=0.3,
    mastery_after=0.6,
    is_correct=True,
    response_time=5.2,  # 可选，不传时使用估算值
)
# 结束学习会话
summary = engine.end_learning_session(user_id=1)
print(summary)
```
## 性能优化建议
1. **缓存策略**: 跻加 Redis 缓存用户-物品矩阵和相似度计算结果
2. **批量计算**: 协同过滤相似度计算可异步进行
3. **A/B 测试**: 为新算法添加开关，便于渐进式发布
4. **监控告警**: 添加推荐效果监控和告警
5. **前端计时**: 在前端实现真实 `response_time` 计时并传给后端，替代当前估算值
6. **性能观测**: 通过 `/api/health/metrics` 监控 DB 查询次数，验证预加载优化效果
