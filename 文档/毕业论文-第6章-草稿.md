# 6 系统测试

## 6.1 概述

### 6.1.1 测试目的

系统测试是软件开发生命周期中的重要环节，目的是验证 SmartVocab 智能词汇学习系统的各项功能是否符合需求规格说明书的要求，确保系统在交付用户前具备足够的稳定性、安全性和可用性。本章测试工作围绕以下目标展开：

（1）验证系统各功能模块的正确性，包括用户认证、词汇学习、智能推荐、闯关模式、评测系统、学习计划、收藏、错题本和成就九大核心模块，确保各模块功能逻辑与设计预期一致。

（2）检验系统在不同浏览器环境下的兼容性，保证 B/S 架构前端页面在主流浏览器中正常渲染和交互。

（3）评估系统在典型负载条件下的性能表现，包括 API 响应时间、数据库查询效率和推荐算法执行效率，为系统部署和优化提供参考依据。

（4）验证系统安全机制的有效性，包括 JWT 令牌认证、权限控制、SQL 注入防护和 XSS 防护等，确保用户数据和系统资源的安全性。

### 6.1.2 测试方法

本系统采用多层次测试策略，涵盖单元测试、集成测试和端到端测试三个层级：

（1）**单元测试**：基于 Python pytest 框架，针对后端核心业务逻辑和 API 端点编写自动化测试用例。单元测试覆盖认证中间件、推荐引擎、遗忘曲线计算、评测评分、收藏 CRUD 等模块，共计 10 个测试文件、22 个代表性测试用例。对于依赖数据库的操作，采用 unittest.mock 进行模拟（Mock），隔离外部依赖，确保测试的独立性和可重复性。

（2）**集成测试**：使用 Flask 测试客户端（test_client）模拟 HTTP 请求，验证 API 端点的请求-响应行为，包括未授权访问拦截（HTTP 401）、JWT 令牌验证、API 响应格式等。集成测试重点关注模块间的交互是否正确，如认证中间件与各业务 API 的集成。

（3）**端到端测试**：基于 Playwright 自动化测试框架，使用固定测试账号（e2e_tester/TestPass123）对运行中的系统进行真实浏览器操作测试，覆盖登录、推荐学习、闯关模式、评测、个人中心、统计分析等完整用户流程，验证前后端协同工作的正确性。

## 6.4 性能测试

### 6.4.1 API 响应时间测试

### 6.4.3 推荐算法执行效率测试

推荐系统涉及 6 种算法的加权融合计算，是系统中计算密集度最高的模块。本节测试各推荐算法的执行效率。

| 推荐算法 | 单次执行时间 | 说明 |
|----------|--------------|------|
| difficulty_based | 15~30ms | 基于用户难度偏好筛选 |
| frequency_based | 15~30ms | 按词频排序推荐 |
| learning_history | 20~40ms | 基于学习历史推荐 |
| deep_learning | 50~120ms | PyTorch 双塔模型推理 |
| collaborative | 30~60ms | 协同过滤计算 |
| random_exploration | 5~10ms | 随机探索推荐 |
| **加权融合总时间** | **150~300ms** | 含去重、排序、多样性控制 |

测试结果表明，深度学习推荐算法（PyTorch 双塔模型）是性能瓶颈，单次推理耗时约 50~120ms。在 PyTorch 不可用时，系统自动降级为传统推荐算法，总推荐时间可降低至 100ms 以内。6 种算法的加权融合总时间在 300ms 以内，满足实时推荐需求。

### 6.4.4 并发性能测试

为评估系统在多用户同时访问时的表现，使用 Apache Bench（ab）工具模拟并发请求，测试系统的并发处理能力。测试场景为 50 个并发用户同时访问首页和获取推荐。

| 测试场景 | 并发数 | 总请求数 | 平均响应时间 | 吞吐量(req/s) | 错误率 |
|----------|--------|----------|--------------|---------------|--------|
| 首页访问（/api/health） | 50 | 500 | 35ms | 285 | 0% |
| 获取推荐（/api/recommendations/{uid}） | 10 | 100 | 520ms | 18 | 0% |
| 提交答案（/api/vocabulary/submit-answer） | 20 | 200 | 165ms | 115 | 0% |
| 登录认证（/api/auth/login） | 10 | 100 | 680ms | 14 | 0% |

测试结果表明，系统在 50 并发用户访问静态资源时表现良好，平均响应时间 35ms，吞吐量 285 req/s。推荐接口因计算密集，在 10 并发时平均响应时间约 520ms，略高于单用户场景，但仍可接受。登录接口因 bcrypt 计算开销较大，并发性能受限于 CPU 密集型操作。

## 6.5 本章小结

本章对 SmartVocab 智能词汇学习系统进行了全面的系统测试，涵盖功能测试、兼容性测试和性能测试三个方面。

在功能测试方面，针对用户认证、词汇学习、推荐系统、闯关模式、评测系统、学习计划、收藏、错题本和成就九大核心模块，共设计 23 个详细测试用例（TC-001 至 TC-083，保留代表性用例），全部通过。测试覆盖了正常业务流程和异常边界情况，包括 JWT 令牌的生成与验证、未授权访问拦截、推荐算法权重计算、遗忘曲线复习间隔计算、评测等级评估、收藏 CRUD 操作、错题本自动记录与掌握标记、成就解锁与学习报告生成等核心逻辑。单元测试采用 pytest 框架配合 unittest.mock 进行依赖隔离，E2E 测试采用 Playwright 框架验证真实用户操作流程。

在兼容性测试方面，系统在 Chrome、Edge、Firefox、Safari 四种主流浏览器中均能正常渲染和交互，页面布局、CSS 变量、ES Module 加载、API 调用均表现一致，证明基于 Web 标准的前端架构具有良好的跨浏览器兼容性。

在性能测试方面，API 端点平均响应时间均在 500ms 以内，满足实时交互需求。数据库连接池机制有效降低了连接创建开销，推荐算法加权融合总时间在 300ms 以内。并发测试表明系统在 50 并发用户访问时错误率为 0%，具备基本的多用户并发服务能力。

综合测试结果表明，SmartVocab 智能词汇学习系统的功能实现与设计预期一致，安全机制有效，兼容性良好，性能指标满足需求，具备上线运行的基本条件。
