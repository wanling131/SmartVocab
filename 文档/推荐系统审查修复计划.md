# 推荐系统审查修复计划

## Context

审查发现 SmartVocab 推荐系统架构设计完善（6 种算法 + 协同过滤 + 深度学习 + 遗忘曲线 + MMR 多样性控制），但存在**关键逻辑断裂**：自适应难度调整和动态权重系统的数据输入端从未被接入学习流程，导致这些高级功能形同虚设。同时存在多个 N+1 查询和重复 DB 查询性能问题。

本计划不修改数据库 Schema，仅修复代码层逻辑和性能问题。

---

## 修复项 1：集成 `record_feedback()` 到答题流程

**问题：** `RecommendationEngine.record_feedback()` 已实现但**从未被调用**。`RealtimePersonalizer` 和 `DynamicWeightAdjuster` 收不到任何数据，自适应难度和动态权重完全无效。

**文件：** `core/vocabulary/vocabulary_learning_manager.py`

**修改位置：** `submit_answer()` 方法（约 line 226-330），在 `invalidate_user_cache(user_id)` 之后（line 283）添加反馈记录调用：

```python
# 记录推荐系统反馈（用于自适应难度和动态权重）
try:
    mastery_before = existing_record["mastery_level"] if existing_record else 0.0
    mastery_after = result.get("new_mastery_level", 0.0)
    self.recommendation_engine.record_feedback(
        user_id=user_id,
        word_id=word_id,
        algorithm_type="mixed",
        mastery_before=mastery_before,
        mastery_after=mastery_after,
        is_correct=is_correct,
    )
except Exception as e:
    logger.debug("推荐反馈记录失败（不影响主流程）: %s", e)
```

**注意：** `response_time` 参数已在 `submit_answer()` 的签名中，但当前未传给 `record_feedback()`。`recommendation_engine.record_feedback()` 内部硬编码了 `response_time = 10 if is_correct else 20`。改为传入实际值更好，但不是必须的（RealtimePersonalizer.update_session 内部可用）。

## 修复项 2：冷启动利用等级测试结果

**问题：** `ColdStartHandler.get_cold_start_recommendations()` 完全不读取 `evaluation_results` 表中的 `assessed_level`，用户测出 B2 级别仍推荐基础词。

**文件：** `core/recommendation/recommendation_enhancements.py`

**修改位置：** `ColdStartHandler.get_cold_start_recommendations()` 方法（约 line 774-809）

**修改内容：**
1. 先查 `evaluation_results` 表获取最新评测结果
2. 根据评测等级映射难度范围（复用 `evaluation_manager.py:19` 的 `LEVEL_TO_DIFFICULTY` 映射）
3. 用映射后的难度范围替代硬编码的 `[1, 2]`

```python
def get_cold_start_recommendations(self, user_id, limit=20):
    # 1. 查等级测试结果
    difficulty_range = self.config["default_difficulty_range"]  # 默认 [1, 2]
    try:
        from tools.evaluation_results_crud import EvaluationResultsCRUD
        eval_crud = EvaluationResultsCRUD()
        eval_history = eval_crud.get_by_user(user_id, limit=1)
        if eval_history:
            assessed = eval_history[0].get("assessed_level", "A2")
            level_map = {"A1": 1, "A2": 1, "B1": 2, "B2": 3, "C1": 4, "C2": 5}
            base_diff = level_map.get(assessed, 1)
            difficulty_range = [max(1, base_diff - 1), min(6, base_diff + 1)]
    except Exception:
        pass

    # 2. 后续逻辑不变，但使用动态 difficulty_range
    recommendations = []
    if self.config["use_popularity"]:
        popular = self._get_popular_words(limit // 2)
        recommendations.extend(popular)
    basic_words = self._get_basic_words(difficulty_range, limit // 2)  # 使用动态范围
    recommendations.extend(basic_words)
    # ... 去重逻辑不变
```

## 修复项 3：推荐引擎预加载共享数据，消除重复 DB 查询

**问题：** `_get_enhanced_mixed_recommendations()` 一次推荐调用 6 个子算法，每个都独立调用 `learning_records_crud.get_by_user()` 和 `words_crud.list_all()`，**一次推荐至少 12 次 DB 查询**，大量重复。

**文件：** `core/recommendation/recommendation_engine.py`

**修改位置：** `_get_enhanced_mixed_recommendations()` 方法（约 line 242-360）和各子算法方法。

**修改方案：** 在 `_get_enhanced_mixed_recommendations()` 入口预加载共享数据，通过参数传递给子算法。

具体改动：
1. 在 `_get_enhanced_mixed_recommendations()` 开头预加载 `all_words` 和 `user_records`（已有 `user_records` 参数）
2. 修改 `_get_difficulty_based_recommendations`、`_get_frequency_based_recommendations`、`_get_history_based_recommendations`、`_get_random_recommendations` 四个方法签名，增加可选参数 `all_words=None`、`user_records=None`
3. 当传入了预加载数据时跳过各自的 DB 查询

```python
def _get_difficulty_based_recommendations(self, user_id, learned_word_ids, limit,

    # 使用预加载数据
    if user_records is None:

    if all_words is None:
        all_words = self.words_crud.list_all(limit=5000)

```

## 修复项 4：遗忘曲线 N+1 查询修复

**问题：** `forgetting_curve_manager._enrich_with_word_details()` 逐个调用 `words_crud.read()`，`words_crud.get_by_ids()` 已实现批量查询但未使用。

**文件：** `core/forgetting_curve/forgetting_curve_manager.py`

**修改位置：** `_enrich_with_word_details()` 方法（约 line 162-195）

**替换为批量查询：**

```python
def _enrich_with_word_details(self, records):
    if not records:
        return []
    word_ids = [r.get("word_id") for r in records if r.get("word_id")]
    if not word_ids:
        return records
    # 批量查询（使用已有的 get_by_ids 方法）
    word_map = {w["id"]: w for w in self.words_crud.get_by_ids(word_ids)}
    return [{**r, **word_map.get(r["word_id"], {})} for r in records]
```

## 修复项 5：学习记录管理器 N+1 查询修复

**问题：** `learning_record_manager.get_user_learning_records()` 逐个调用 `words_crud.read()`，同样可用 `get_by_ids()` 批量替代。

**文件：** `core/learning/learning_record_manager.py`

**修改位置：** `get_user_learning_records()` 方法（约 line 93-146）

**替换批量查询：**

```python
def get_user_learning_records(self, user_id, limit=None, offset=0):
    records = self.learning_records_crud.get_by_user(user_id, limit, offset)
    if not records:
        return []

    # 批量获取单词详情
    word_ids = [r["word_id"] for r in records]
    word_map = {w["id"]: w for w in self.words_crud.get_by_ids(word_ids)}

    enriched_records = []
    for record in records:
        word_info = word_map.get(record["word_id"])
        if word_info:
            enriched_record = {
                **record,
                "word": word_info.get("word", ""),
                "translation": word_info.get("translation", ""),
                "phonetic": word_info.get("phonetic", ""),
                "pos": word_info.get("pos", ""),
                "difficulty_level": word_info.get("difficulty_level", 1),
                "is_correct": 1 if record["mastery_level"] >= 0.1 else 0,
                "created_at": record["last_reviewed_at"],
            }
        else:
            enriched_record = {
                **record,
                "word": f"未知单词(ID:{record['word_id']})",
                "translation": "单词信息缺失",
                "phonetic": "",
                "pos": "",
                "difficulty_level": 1,
                "is_correct": 1 if record["mastery_level"] >= 0.1 else 0,
                "created_at": record["last_reviewed_at"],
            }
        enriched_records.append(enriched_record)
    return enriched_records
```
