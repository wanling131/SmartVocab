系统的启动入口为 `main.py`，通过命令 `python main.py` 即可启动开发服务器，默认监听端口 5000。同时提供了 `start.bat` 一键启动脚本和 `test.bat` 一键测试脚本，便于开发调试。

## 5.2 系统层次架构

SmartVocab 系统采用经典的三层架构设计，前后端分离部署，各层职责明确、耦合度低。整体层次架构如图 5-1 所示。

[图5-1 系统层次架构图]

### 5.2.1 后端三层架构

后端按照职责划分为三个层次：API 接口层、核心业务层和数据访问层。

**（1）API 接口层（api/）**

API 接口层由 10 个 Flask Blueprint 组成，负责接收 HTTP 请求、参数校验、权限控制和响应封装。每个 Blueprint 对应一个功能模块，通过统一的 URL 前缀进行路由隔离。表 5-2 列出了系统中的 10 个蓝图及其职责。

**表 5-2 后端 API 蓝图列表**

| 蓝图名称 | URL 前缀 | 功能描述 |
|----------|----------|----------|
| health_bp | /api | 健康检查与系统监控 |
| auth_bp | /api/auth | 用户注册、登录、个人信息管理 |
| vocabulary_bp | /api/vocabulary | 词汇学习、会话管理、词库管理 |
| learning_bp | /api/learning | 学习记录、进度统计、遗忘曲线 |
| recommendation_bp | /api/recommendations | 智能推荐算法 |
| plans_bp | /api/plans | 学习计划管理 |
| evaluation_bp | /api/evaluation | 等级评测 |
| levels_bp | /api/levels | 闯关模式与关卡管理 |
| achievements_bp | /api/achievements | 成就系统与学习报告 |
| favorites_bp | /api/favorites | 收藏管理 |

该层通过 `auth_middleware.py` 提供统一的认证与授权机制。`@require_auth` 装饰器对所有需要登录的接口进行 JWT 令牌验证，`check_user_access()` 函数确保用户只能访问自己的数据，防止越权访问。所有接口通过 `APIResponse` 类统一封装响应格式，包含 `success`、`message`、`data` 三个标准字段，保证前端能够以一致的方式解析后端响应。

API 启动器 `api_launcher.py` 负责组装所有蓝图，同时配置 CORS 跨域策略、请求追踪中间件（生成唯一请求 ID 并记录响应耗时）、全局异常处理和安全响应头（X-Content-Type-Options、X-Frame-Options 等）。

**（2）核心业务层（core/）**

核心业务层封装了系统的核心业务逻辑，包括 7 个核心管理器：

- `UserAuth`（用户认证）：基于 bcrypt 的密码哈希与验证，用户注册、登录、信息管理
- `RecommendationEngine`（推荐引擎）：6 种推荐算法的混合调度，动态权重调整
- `DeepLearningRecommendationEngine`（深度学习推荐）：PyTorch 双塔神经网络推荐
- `VocabularyLearningManager`（词汇学习）：学习会话管理、题目生成、答案评判
- `LearningRecordManager`（学习记录）：学习记录的 CRUD 与掌握度更新
- `ForgettingCurveManager`（遗忘曲线）：艾宾浩斯遗忘曲线计算与复习调度
- `EvaluationManager`（评测管理）：评测试卷生成与评分

以推荐引擎为例，其内部实现了难度匹配、词频排序、学习历史分析、深度学习预测、协同过滤和随机探索共 6 种推荐策略，并集成了冷启动处理、动态权重调整、多样性控制（MMR 算法）和实时个性化 4 个增强模块，构成完整的混合推荐系统。

**（3）数据访问层（tools/）**

数据访问层通过 `BaseCRUD` 基类提供统一的数据库操作接口，所有表级 CRUD 类继承该基类。系统共包含 14 个表级 CRUD 类，覆盖用户、单词、学习记录、学习会话、推荐结果、收藏、成就、关卡进度等全部数据表。

`DatabaseManager` 以单例模式管理 MySQL 连接池，默认池大小为 10，通过 `get_database_context()` 上下文管理器自动获取和释放连接。`BaseCRUD` 类提供三个核心操作方法：`execute_query()` 执行查询、`execute_update()` 执行更新、`execute_insert()` 执行插入，所有方法均使用参数化查询防止 SQL 注入。`build_update_query()` 方法通过 `allowed_fields` 白名单机制过滤可更新字段，防止恶意字段注入。

此外，`memory_cache.py` 提供了基于 TTL 过期和 LRU 淘汰策略的内存缓存，包含单词缓存（10 分钟 TTL）、用户记录缓存（2 分钟 TTL）、推荐结果缓存（5 分钟 TTL）等 6 个全局缓存实例，有效减少数据库查询次数。缓存的键通过函数生成器（如 `make_recommendation_key()`）统一管理，支持用户级缓存批量失效。

### 5.2.2 前端三层架构

前端采用多页架构，由 9 个独立 HTML 页面组成，按照职责划分为视图层、组件层和通信层。

**（1）视图层（pages/）**

视图层包含 9 个 HTML 页面，分别是登录页（login.html）、仪表盘（dashboard.html）、学习页（learning.html）、统计页（statistics.html）、学习计划页（plans.html）、闯关页（levels.html）、评测页（evaluation.html）、收藏页（favorites.html）和个人资料页（profile.html）。每个页面独立加载，通过 ES Module 导入共享的 JavaScript 模块。

**（2）组件层（js/）**

组件层提供了可复用的前端组件和工具函数。`page-common.js` 封装了导航栏渲染和用户状态管理；`utils.js` 提供了 `escapeHtml()`（XSS 防护）、`showToast()`（消息提示）、`animateNumber()`（数字动画）等通用工具函数；`charts.js` 负责统计图表渲染；`components/achievements.js` 实现了成就徽章组件；`icons.js` 提供了与克莱因蓝主题一致的 SVG 图标系统。

**（3）通信层（js/api-client.js）**

通信层封装了与后端 API 的所有交互逻辑，核心函数 `apiRequest()` 负责发起 HTTP 请求并处理 JWT 令牌的自动附加。该模块实现了三层性能优化机制：

- **GET 请求缓存**：对 GET 请求结果进行 2 分钟 TTL 缓存，避免重复请求
- **请求去重**：通过 `pendingRequests` Map 检测并合并同时发出的相同请求
- **缓存预热**：登录成功后通过 `warmupUserCache()` 并行预加载用户常用数据

此外，`fetchWithState()` 函数在 `apiRequest()` 基础上集成了加载状态管理和错误提示，`userApi` 对象则提供了学习进度、统计、收藏、推荐等常用 API 的一站式调用封装。

## 5.3 功能模块实现

### 5.3.1 系统主页实现

系统主页即仪表盘页面（dashboard.html），是用户登录后看到的第一个页面，承载着学习概览、推荐词汇和进度统计三大核心功能区域。

[图5-2 系统仪表盘主页]

**（1）学习概览区域**

页面顶部展示用户的学习概览信息，包括已学单词数、已掌握单词数、今日学习量等关键指标，这些数据通过 `/api/learning/progress/<uid>` 接口获取。数字展示采用 `animateNumber()` 动画函数，以缓出（ease-out）的效果递增显示，提升视觉体验。

**（2）智能推荐区域**

页面中部展示推荐词汇列表，系统通过推荐算法为用户精选适合当前水平的单词。推荐词汇以卡片形式排列，每张卡片显示单词、音标、难度等级和推荐理由。推荐理由由后端根据算法类型自动生成，如"AI 推荐：高频核心词汇"、"基础词汇：适合当前学习阶段"等。前端通过 `/api/recommendations/<uid>` 接口获取推荐数据，该接口支持分页加载和刷新机制。

**（3）进度统计区域**

页面底部展示学习进度的可视化图表，包括词汇掌握分布和学习趋势曲线，通过 Chart.js 渲染。图表数据从 `/api/learning/statistics/<uid>` 接口获取。

仪表盘页面的核心数据加载逻辑如下所示：

```javascript
// 并行加载用户常用数据（缓存预热）
export async function warmupUserCache(userId) {
  const endpoints = [
    `/learning/progress/${userId}`,
    `/learning/statistics/${userId}`,
    `/recommendations/${userId}?limit=50`,
    `/favorites/${userId}/ids`
  ]
  Promise.all(
    endpoints.map(ep => apiRequest(ep, { useCache: true }).catch(() => null))
  )
}
```

上述代码展示了登录后的缓存预热机制。通过 `Promise.all` 并行请求 4 个关键接口，利用 `useCache: true` 选项将结果写入前端缓存，后续页面访问可直接从缓存读取，显著提升了页面加载速度。

### 5.3.2 用户认证实现

用户认证模块实现了基于 JWT 的无状态认证机制，涵盖用户注册、登录、令牌验证和密码修改四个核心功能。

[图5-3 用户登录页面]

**（1）注册流程**

用户在登录页填写用户名和密码后，前端通过 `apiRequest('/auth/register', { method: 'POST', body: ... })` 发送注册请求。后端 `auth_bp` 蓝图接收请求后调用 `UserAuth.register()` 方法进行注册处理。注册流程包括用户名合法性校验（长度 2-50 字符）、密码强度校验（长度 6-200 字符）、用户名唯一性检查，最后使用 bcrypt 算法（rounds=12）对密码进行哈希处理后存入数据库。注册成功后自动生成 JWT 令牌返回前端。

注册接口的核心实现如下：

```python
# api/auth_api.py
@auth_bp.route("/register", methods=["POST"])
@handle_api_error
def register():
    data = request.get_json()
    username = data.get("username")
    password = data.get("password")
    email = data.get("email")

    if not username or not password:
        return APIResponse.error("用户名和密码不能为空", 400)

    result = user_auth.register(username, password, email)
    if result["success"]:
        token = generate_token(result.get("user_id"), username)
        return APIResponse.success(
            {"user_id": result.get("user_id"), "username": username, "token": token},
            result["message"],
        )
    else:
        return APIResponse.error(result["message"], 400)
```

**（2）登录与 JWT 令牌**

用户登录时，系统从数据库查询用户记录，使用 `bcrypt.checkpw()` 验证密码哈希。验证通过后，调用 `generate_token()` 生成 JWT 令牌。令牌的 Payload 包含 `user_id`、`username`、`exp`（过期时间，默认 24 小时）和 `iat`（签发时间）四个字段，使用 HS256 算法签名。

令牌生成与验证的核心代码如下：

```python
# api/auth_middleware.py
def generate_token(user_id: int, username: str) -> str:
    now = datetime.now(timezone.utc)
    payload = {
        "user_id": user_id,
        "username": username,
        "exp": now + timedelta(hours=JWT_EXPIRATION_HOURS),
        "iat": now,
    }
    return jwt.encode(payload, JWT_SECRET_KEY, algorithm=JWT_ALGORITHM)

def verify_token(token: str) -> dict:
    try:
        payload = jwt.decode(token, JWT_SECRET_KEY, algorithms=[JWT_ALGORITHM])
        return payload
    except jwt.ExpiredSignatureError:
        return None
    except jwt.InvalidTokenError:
        return None
```

前端接收到令牌后存储在 `localStorage` 的 `auth_token` 键中，后续所有 API 请求通过 `Authorization: Bearer <token>` 请求头自动携带令牌。`apiRequest()` 函数在每次请求时自动从 `localStorage` 读取令牌并附加到请求头中。

**（3）权限控制**

系统实现了细粒度的权限控制机制。`@require_auth` 装饰器用于保护需要登录的接口，从请求头解析 JWT 令牌，验证失败返回 401 状态码。`check_user_access(user_id)` 函数验证当前用户是否有权访问目标用户数据，防止横向越权攻击。管理员操作（如词库导入、单词 CRUD）通过 `ADMIN_USERS` 环境变量控制，仅允许指定的管理员用户名执行。生产环境下强制要求 JWT 密钥至少 32 个字符，拒绝使用弱密钥。

### 5.3.3 词汇学习实现

词汇学习模块是系统的核心功能模块，实现了完整的学习会话流程：推荐选词、题目生成、答题评判、掌握度更新和遗忘曲线记录。

[图5-4 词汇学习页面]

**（1）学习会话流程**

学习会话由 `VocabularyLearningManager` 类管理，完整流程如下：

1. **开始会话**：用户进入学习页面后，前端调用 `/api/vocabulary/start-session` 接口。后端根据是否指定难度等级，调用推荐引擎获取推荐词汇或按难度筛选词汇，创建学习会话并保存到数据库。

```python
# core/vocabulary/vocabulary_learning_manager.py
def start_learning_session(self, user_id, difficulty_level=None,
                          word_count=20, question_type="mixed",
                          first_word_id=None):
    # 获取推荐单词
    if difficulty_level:
        recommendations = self._get_words_by_difficulty(difficulty_level, word_count)
    else:
        recommendations = self.recommendation_engine.get_recommendations(
            user_id, word_count)

    # 创建学习会话
    session_info = {
        "user_id": user_id,
        "words": recommendations,
        "current_word_index": 0,
        "correct_count": 0,
        "total_count": len(recommendations),
        "start_time": datetime.now(),
        "session_type": "learning",
        "question_type": question_type,
        "word_stages": {},
    }
    session_id = self.learning_sessions_crud.create(user_id, session_info, "learning")
    session_info["session_id"] = session_id
    return {"success": True, "session_info": session_info}
```

2. **获取当前题目**：前端调用 `/api/vocabulary/current-word` 接口获取当前单词的题目。系统根据题型设置（choice/translation/spelling/mixed）生成对应的题目。在混合模式下，同一单词先展示选择题，答完后再展示翻译题，通过 `word_stages` 字段记录每个单词的当前学习阶段。

3. **提交答案**：用户提交答案后，系统调用 `_check_answer()` 方法进行答案判定。选择题采用精确匹配，拼写题不区分大小写匹配，翻译题采用智能匹配（支持完全匹配、主要意思匹配、包含关系匹配三种策略）。

4. **更新掌握度**：答案提交后更新学习记录。对于已有记录，根据答题结果调整掌握度；对于新单词，根据题目类型设置初始掌握度（选择题答对 0.2、翻译题答对 0.3、拼写题答对 0.4）。同时将答题反馈记录到推荐系统，用于动态权重调整。

**（2）题目生成机制**

系统支持三种题型的自动生成：

- **选择题**：从数据库获取 3 个随机单词作为干扰项，与正确答案合并后打乱顺序。答案选项经过词性前缀处理（如"n. 时间"），提高题目区分度。
- **翻译题**：展示英文单词和音标，用户输入中文释义，系统进行智能匹配判定。
- **拼写题**：展示中文释义和音标，用户输入英文单词，不区分大小写精确匹配。

选择题生成的核心逻辑如下：

```python
# core/vocabulary/vocabulary_learning_manager.py
def _generate_choice_question(self, word_info):
    correct_translation = self._simplify_translation(
        word_info["translation"], word_info["pos"])
    wrong_options = self._generate_wrong_options(
        correct_translation, word_info["id"])
    all_options = [correct_translation] + wrong_options
    random.shuffle(all_options)
    return {
        "word_id": word_info["id"],
        "word": word_info["word"],
        "phonetic": word_info["phonetic"],
        "question_type": "choice",
        "question": f"请选择单词 '{word_info['word']}' 的正确中文释义：",
        "options": all_options,
        "correct_answer": correct_translation,
    }
```

**（3）错题本功能**

用户答错时，系统自动将错题记录到 `mistake_book` 表。用户可以在错题本页面查看历史错题，并通过 `/api/vocabulary/mistakes/<uid>/review` 接口获取错题进行专项复习。复习答对后可通过 `/api/vocabulary/mistakes/<uid>/word/<wid>/master` 接口标记为已掌握。

### 5.3.4 智能推荐实现

智能推荐模块是系统的核心创新点，实现了基于 6 种推荐算法的混合推荐策略，并集成了动态权重调整、多样性控制、冷启动处理和实时个性化四项增强机制。

[图5-5 智能推荐词汇展示页面]

**（1）混合推荐算法框架**

推荐引擎（`RecommendationEngine`）采用"多算法并行打分、加权融合排序"的策略。系统配置了 6 种推荐算法及其基础权重，如表 5-3 所示。

**表 5-3 推荐算法权重配置**

| 算法名称 | 权重 | 描述 |
|----------|------|------|
| difficulty_based | 0.25 | 基于难度匹配的推荐 |
| frequency_based | 0.20 | 基于词频排序的推荐 |
| deep_learning | 0.20 | 基于深度学习模型的推荐 |
| random_exploration | 0.15 | 随机探索推荐 |
| learning_history | 0.15 | 基于学习历史的推荐 |
| collaborative | 0.05 | 基于协同过滤的推荐 |

在混合推荐模式下，各算法独立生成候选列表，然后按权重计算综合推荐分数。每个单词的最终分数 = sum(各算法权重 * 该算法给出的分数)。系统通过 `_get_enhanced_mixed_recommendations()` 方法实现这一过程。

权重配置存储在 `config.py` 的 `RECOMMENDATION_CONFIG` 字典中，支持运行时调整而无需修改代码。

**（2）六种推荐算法实现**

**基于难度的推荐**：根据用户当前平均掌握程度推算目标难度等级（`target_difficulty = avg_mastery * 6 + 1`），从该难度的未学单词中随机选取。新用户默认推荐难度等级 1 的单词。

**基于词频的推荐**：从适合用户难度的单词中，按 `frequency_rank` 字段升序排列，优先推荐高频词汇。词频排名前 100 的单词推荐分数为 1.0，并随排名递减。

**基于学习历史的推荐**：分析用户的学习偏好（偏好难度、学习速度等），计算候选单词与用户偏好的相似度分数，综合考虑难度匹配度和词频权重。

**深度学习推荐**：使用 PyTorch 实现的双塔神经网络模型（详见下文），从用户-单词交互数据中学习个性化推荐模式。

**协同过滤推荐**：同时使用基于用户和基于物品的协同过滤。基于用户 CF 找到学习偏好相似的用户，推荐他们学过但当前用户未学的单词；基于物品 CF 根据单词共现关系推荐相关单词。

**随机探索推荐**：在用户难度范围内的未学单词中随机选取，推荐分数随机生成（0.1-0.5），用于增加推荐多样性，避免"信息茧房"效应。

**（3）深度学习推荐模型**

深度学习推荐器采用双塔（Dual-Tower）架构，包含单词编码器和用户编码器两个独立的神经网络塔，以及交叉注意力融合机制。模型结构如图 5-6 所示。

[图5-6 双塔神经网络模型架构]

单词编码器和用户编码器均采用两层全连接网络结构（128 -> 64），每层使用 LayerNorm 归一化、ReLU 激活和 0.2 的 Dropout 正则化。编码器将 25 维的原始特征映射到 64 维的隐空间表示。

```python
# core/recommendation/deep_learning_recommender.py
class DeepLearningRecommender(nn.Module):
    def __init__(self, word_feature_dim, user_feature_dim,
                 hidden_dims=[128, 64, 32], use_attention=True):
        super().__init__()
        # 单词特征编码器
        self.word_encoder = nn.Sequential(
            nn.Linear(word_feature_dim, hidden_dims[0]),
            nn.LayerNorm(hidden_dims[0]),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(hidden_dims[0], hidden_dims[1]),
            nn.LayerNorm(hidden_dims[1]),
            nn.ReLU(),
            nn.Dropout(0.2),
        )
        # 用户特征编码器（结构相同）
        self.user_encoder = nn.Sequential(
            nn.Linear(user_feature_dim, hidden_dims[0]),
            nn.LayerNorm(hidden_dims[0]),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(hidden_dims[0], hidden_dims[1]),
            nn.LayerNorm(hidden_dims[1]),
            nn.ReLU(),
            nn.Dropout(0.2),
        )
        # 交叉注意力机制
        if use_attention:
            self.attention = nn.MultiheadAttention(
                embed_dim=hidden_dims[1], num_heads=4, dropout=0.1,
                batch_first=True)
            self.context_gate = nn.Sequential(
                nn.Linear(hidden_dims[1] * 3, hidden_dims[1]), nn.Sigmoid())
        # 融合层
        fusion_input_dim = hidden_dims[1] * 3 if use_attention else hidden_dims[1] * 2
        self.fusion = nn.Sequential(
            nn.Linear(fusion_input_dim, hidden_dims[2]),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(hidden_dims[2], 1),
            nn.Sigmoid(),
        )
```

模型的前向传播过程如下：首先分别通过单词编码器和用户编码器获取 64 维的隐空间表示；然后通过多头交叉注意力机制（4 头）计算用户对单词特征和单词对用户特征的注意力权重，捕获特征间的交互关系；接着通过门控融合机制将原始编码和注意力增强后的表示进行加权融合；最后通过全连接融合层输出 0-1 之间的推荐分数。

模型的输入特征维度为 25 维，单词特征包括难度等级、词频排名、词性、CEFR 等级、口语/学术比例等；用户特征包括平均掌握度、学习速度、偏好难度等。系统在用户积累 50 条学习记录后启动模型训练，使用 MSE 损失函数和 Adam 优化器，默认训练 50 个 Epoch。

当 PyTorch 不可用时，系统自动降级为传统推荐算法，保证服务的可用性。

**（4）推荐增强机制**

系统集成了四项推荐增强机制：

- **冷启动处理**：新用户（学习记录少于 15 条）使用热门词推荐和评测结果初始化策略
- **动态权重调整**：`DynamicWeightAdjuster` 根据用户对各算法推荐的反馈（答题正确率、掌握度变化）动态调整算法权重
- **多样性控制**：使用 MMR（Maximal Marginal Relevance）算法平衡推荐的相关性和多样性，限制同词性单词的最大比例（不超过 40%）
- **实时个性化**：`RealtimePersonalizer` 根据当前学习会话的答题表现实时调整推荐策略，近期行为权重为 0.4，当前会话权重为 0.3

**（5）推荐分页与缓存**

推荐接口采用"池化分页"策略：首次请求时计算 50 个推荐词作为推荐池缓存到内存，前端每次请求取 5 个，共可翻 10 页。翻到最后一页时自动重新计算新的推荐池并重置页码。推荐结果在后端通过 `recommendation_cache`（5 分钟 TTL）缓存，在前端通过 `apiCache`（2 分钟 TTL）双重缓存，大幅减少重复计算。

### 5.3.5 闯关模式实现

闯关模式（levels 模块）为词汇学习增加了游戏化元素，用户按照预设的关卡序列逐级挑战，每个关卡对应特定的难度等级和词汇集合。

**（1）关卡数据结构**

关卡信息存储在 `level_gates` 数据表中，包含关卡名称（`gate_name`）、关卡顺序（`gate_order`）、难度等级（`difficulty_level`）和单词数量（`word_count`）等字段。用户的通关进度存储在 `user_level_progress` 表中，通过 `level_gate_id` 关联关卡，记录解锁状态、完成状态、掌握数量和最佳分数。

由于前端与数据库字段名存在差异（如前端期望 `name` 而数据库为 `gate_name`），API 层在返回数据时添加字段别名以保持兼容：

**（2）关卡解锁逻辑**

关卡采用线性解锁机制：第一关默认解锁，后续关卡需要完成前一关才能解锁。完成条件为正确率达到 80% 以上。通关成功后自动解锁下一关并创建进度记录。

解锁接口的核心实现如下：

```python

@levels_bp.route("/unlock", methods=["POST"])
@require_auth
def unlock_gate():
    data = request.get_json()
    user_id = data.get("user_id")
    level_gate_id = data.get("level_gate_id")
    # 检查上一关是否完成
    all_gates = gates_crud.list_all()
    prev_gate = next(
        (g for g in all_gates if g["gate_order"] == gate["gate_order"] - 1), None)
    if prev_gate:
        prev_progress = progress_crud.get_by_user_gate(user_id, prev_gate["id"])
        if prev_progress and not prev_progress.get("is_completed"):
            return APIResponse.error("请先完成上一关", 400)
    success = progress_crud.unlock_gate(user_id, level_gate_id)
    return APIResponse.success(updated, "解锁成功")
```

**（3）闯关学习会话**

用户进入关卡后，系统根据关卡难度从词库中选取对应难度的单词，优先选择未学习过的单词。如果可用单词不足，则从已学习的同难度单词中补充。学习会话采用混合题型（选择题+翻译题），会话数据保存到 `learning_sessions` 表中，类型标记为"gate"。

### 5.3.6 评测系统实现

评测系统（evaluation 模块）为用户提供词汇水平等级测试，通过标准化测试流程评估用户的词汇掌握水平。

[图5-8 等级评测页面]

**（1）评测流程**

评测系统实现了"生成试卷 -> 答题 -> 自动评分 -> 生成报告"的完整流程：

1. **开始评测**：用户选择评测参数后，前端调用 `/api/evaluation/start` 接口。`EvaluationManager` 根据参数从词库中抽取指定数量的单词，自动生成选择题格式的试卷，每道题包含 4 个选项。试卷信息保存到 `evaluation_papers` 表中。

评测题目使用选择题格式，保证评测的客观性和一致性。每个单词生成一道四选一选择题，干扰项从数据库中随机选取。评测可按难度等级或词库类型（如 CET4、CET6）进行筛选，支持定制化评测。

### 5.3.9 统计与复习实现

统计与复习模块（learning 模块）为用户提供多维度的学习数据可视化和基于遗忘曲线的智能复习调度。

[图5-12 学习统计页面]

**（1）学习统计**

统计页面通过图表可视化展示用户的学习数据，包括学习进度概览、学习趋势曲线、词汇掌握分布和难度分布等。数据通过 `/api/learning/statistics/<uid>` 和 `/api/learning/progress/<uid>` 接口获取。

`LearningRecordManager` 的 `get_learning_statistics()` 方法支持按天数范围查询统计数据，默认统计最近 7 天。`get_learning_progress()` 方法则返回用户的总体学习进度，包括已学单词总数、已掌握单词数、平均掌握度等指标。

**（2）遗忘曲线复习**

遗忘曲线模块基于艾宾浩斯遗忘曲线理论，实现智能复习调度。核心由 `ForgettingCurveManager` 类管理。

复习间隔的计算公式为：

```
interval = base_interval * mastery_factor * review_factor
```

其中：
- `base_interval`：基础间隔 0.5 小时
- `mastery_factor = 1 + mastery_level * 3.0`：掌握程度因子，掌握度越高间隔越长
- `review_factor = 2^min(review_count, 10)`：复习次数因子，指数增长（最大 2^10）

最终间隔限制在 1 小时到 30 天之间。记忆保持率的预测公式为：

```python
strength = 1 + mastery_level * 10
retention_rate = exp(-hours_since_review / strength)
```

复习调度的核心实现如下：

```python
# core/forgetting_curve/forgetting_curve_manager.py
def _calculate_review_interval(self, mastery_level, review_count):
    base_interval = self.base_interval
    mastery_factor = 1 + (mastery_level * self.mastery_coefficient)
    review_factor = math.pow(
        2, min(review_count, LEARNING_PARAMS["max_review_count"]))
    interval = base_interval * mastery_factor * review_factor
    min_interval = 1
    max_interval = 24 * 30
    return max(min_interval, min(interval, max_interval))
```

**（3）复习单词获取**

获取待复习单词时，系统优先从数据库的 `next_review_at` 字段筛选到期词汇，按紧急程度排序。掌握度低于 0.6 的单词标记为最高紧急度（1000），已过期的单词根据超时时长计算紧急度，未过期但到期的单词使用默认紧急度（500）。系统通过批量查询单词详情（`get_by_ids()`）替代逐条查询，避免 N+1 查询问题。

复习完成后，系统更新学习记录的 `mastery_level`、`review_count`、`last_reviewed_at` 和 `next_review_at` 字段。答对时掌握度增加 0.15-0.25（响应时间越短增幅越大），答错时掌握度减少 0.1-0.2（思考时间越长减幅越小），体现了"快速准确回忆表明记忆更牢固"的认知科学原理。

## 5.4 本章小结

本章详细阐述了 SmartVocab 智能词汇学习系统的实现过程。系统采用后端三层（API 接口层、核心业务层、数据访问层）和前端三层（视图层、组件层、通信层）的架构设计，通过 10 个 Flask Blueprint 提供 RESTful JSON API 服务，前端以 9 个独立 HTML 页面组成多页应用。

在安全方面，系统实现了 JWT 无状态认证、bcrypt 密码哈希、参数化查询、白名单字段过滤和 XSS 防护等多层安全机制。在推荐算法方面，系统集成了 6 种推荐算法的混合调度框架，其中深度学习分支基于 PyTorch 双塔神经网络与交叉注意力机制，配合动态权重调整、MMR 多样性控制和实时个性化等增强策略。在性能优化方面，系统采用了 MySQL 连接池、多层内存缓存、前端请求缓存与去重、推荐池化分页等策略，有效保障了系统的响应速度。智能复习模块基于艾宾浩斯遗忘曲线实现自适应间隔调度，综合考虑掌握程度和复习次数动态调整复习节奏。
