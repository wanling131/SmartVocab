# SmartVocab 开发与部署说明

## 环境变量

复制项目根目录 `.env.example` 为 `.env`，配置 MySQL、日志、CORS、`SECRET_KEY` 等（详见 README 表格）。

**生产环境要点：**

- `APP_ENV=production`（或 `APP_PRODUCTION=true`）
- `SECRET_KEY` 必须设置；`APP_DEBUG=false`；`EXPOSE_ERROR_DETAILS=false`
- `CORS_ORIGINS` 设为前端实际 Origin（勿长期使用 `*`）

## 日志

- 应用启动时由 `main.py` 调用 `config.configure_logging()`。
- 环境变量 `LOG_LEVEL` 可取 `DEBUG`、`INFO`、`WARNING`、`ERROR`。
- 业务代码请使用 `logging.getLogger(__name__)`，避免在生产环境大量 `print`。

## 数据库迁移

首次部署或升级结构时，请执行：

- `文档/数据库建表脚本.sql`
- `文档/数据库升级迁移脚本.sql`

或使用 `python tools/migrate_db.py`（若本地已配置 MySQL）。

## 健康检查

| 接口 | 说明 |
|------|------|
| `GET /api/health` | 进程存活，不连数据库 |
| `GET /api/health/db` | 尝试执行 `SELECT 1`，用于检查数据库 |

可用于负载均衡探活或答辩演示。

## 自动化自检（可选）

项目根目录下 `automation/` 内为答辩前可运行的 API 自检脚本，与主程序无关；**不需要时可整体删除该文件夹**。用法见其中 `README.md`。

## 前端入口

- 使用 ES 模块：`frontend/main.js` 引用 `frontend/js/api-client.js`。
- 静态资源由 Flask `api_launcher` 的 `static_folder='../frontend'` 提供。
