## 1. 基本信息管理

| 设计点 | 实现情况 | 位置 |
|--------|----------|------|
| 词库批量导入 | ✅ | `POST /api/vocabulary/import` |
| 词库导出 | ✅ | `GET /api/vocabulary/export` |
| 单条查询/更新/删除词汇（管理端） | ⚠️ | `tools/words_crud.py` 有方法，**未**暴露独立 REST 给前端 |

| 个人信息查询/修改 | ⚠️ | `GET/PUT /api/auth/profile/<id>` **已有**；前端**无**资料编辑页 |

| 会话管理 | ⚠️ | 前端 `localStorage` 存用户；**无**服务端 JWT/Session 强校验 |
| 密码重置 | ❌ | 未实现接口与页面 |
| CEFR/难度/多套词库 | ⚠️ | 数据字段与选题逻辑支持；无单独「筛选器」管理 UI |

## 代码与联调建议

1. **依赖**：`pip install -r requirements.txt`（含 `werkzeug<3` 与 Flask 2.3 兼容）。
2. **自检**：`python automation/smoke_check.py --quick`（无库）；全量前保证 MySQL 已迁移。
3. **前端错误信息**：已修正 `api-client.js`，HTTP 4xx/5xx 时会优先展示后端 JSON 中的 `message`。

## 答辩表述建议

- 核心学习闭环（注册登录 → 推荐/计划 → 学习/复习 → 评测 → 统计/遗忘曲线）**已打通**。
