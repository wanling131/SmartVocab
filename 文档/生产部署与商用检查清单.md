# SmartVocab 生产部署与商用检查清单

面向正式上线与商业化运营前的自检项（按需取舍）。

## 1. 配置与安全

| 项 | 说明 |
|----|------|
| `APP_ENV=production` | 启用生产模式：默认关闭调试、收紧错误详情 |
| `SECRET_KEY` | **必须**设置强随机密钥（如 `openssl rand -hex 32`），勿提交到仓库 |
| `APP_DEBUG=false` | 关闭 Flask debug，避免代码执行器泄露 |
| `EXPOSE_ERROR_DETAILS=false` | API 不向客户端返回异常堆栈（默认在生产关闭） |
| `CORS_ORIGINS` | 生产环境设为前端实际 Origin，逗号分隔，**勿使用 `*`**（若需 Cookie 跨域） |
| `ENABLE_HSTS=1` | 仅在**全站 HTTPS** 且经反向代理终止 TLS 时启用 |
| `DB_*` | 数据库使用强密码、最小权限账号、网络隔离（仅应用可连） |

## 2. 网络与 TLS

- 生产环境由 **Nginx / Caddy / 云负载均衡** 提供 **HTTPS**，后端可仅监听内网。
- 防火墙仅开放 80/443（及对运维的 SSH 等）。

## 3. 进程与扩展

- 开发：`python main.py`
- 生产：使用 **Gunicorn**（已写入 `requirements.txt`），示例：

```bash
gunicorn -w 4 -b 0.0.0.0:5000 --timeout 120 wsgi:app
```

- 或使用本仓库 **Dockerfile** / **docker-compose.yml**（需先执行数据库迁移脚本）。

## 4. 数据与合规

- 定期备份 MySQL；敏感字段（密码）仅存哈希（bcrypt）。
- 若面向欧盟/国内个人信息合规，需补充隐私政策、用户协议与数据留存策略（法务）。

## 5. 运维与可观测性

- 日志：统一 `LOG_LEVEL`，集中收集（如 ELK、云日志）。
- 探活：`GET /api/health`、`GET /api/health/db` 接入监控告警。
- 资源：关注 PyTorch/大词库场景的内存与 CPU；必要时 API 与训练任务拆分部署。

## 6. 商用增值（代码外）

- 支付、会员、发票、客服、SLA 等需产品与法务配合，不在本仓库范围内。

## 7. 建议后续加固（可选）

- 登录/注册 **限流**（防暴力破解），如 Nginx `limit_req` 或应用层中间件。
- **JWT / Session** 与权限模型（多租户、管理员后台）。
- 静态资源 **CDN** 与前端 **CSP** 策略细化。
