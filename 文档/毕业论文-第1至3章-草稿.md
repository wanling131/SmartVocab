## 摘要

随着在线教育的发展，英语词汇类学习工具在普及的同时，仍面临个性化不足、推荐手段单一、复习节奏与个体记忆规律衔接不紧等问题。本文围绕 **SmartVocab 智能词汇学习系统**的设计与实现展开：融合深度学习与艾宾浩斯遗忘曲线相关思路，采用 **PyTorch** 构建双塔神经网络推荐分支，约定 **20** 维单词特征与 **15** 维用户特征；推荐引擎采用多策略加权融合（深度学习、难度匹配、词频、学习历史、随机探索等，权重以实现与配置为准）。系统提供新词学习与复习会话、等级评测与闯关、学习计划与学习统计等能力，基于 **B/S 架构**与 **RESTful JSON API**，数据持久化采用 **MySQL**。当深度学习分支不可用或模型缺失时，推荐引擎**降级**为传统策略，保证主学习流程可用。全文第 1～3 章对应绪论、关键技术与需求分析，为后续系统分析与设计、实现与测试章节奠定基础。

**关键词**：智能推荐；深度学习；艾宾浩斯遗忘曲线；个性化学习；英语词汇学习；双塔神经网络

**Abstract**

*With the growth of online education, vocabulary learning tools still face challenges such as limited personalization and coarse review scheduling. This thesis presents the design and implementation of SmartVocab, integrating a PyTorch-based dual-tower recommendation branch with multi-strategy fusion and Ebbinghaus-style forgetting-curve–oriented statistics. The system adopts a B/S architecture and RESTful JSON APIs with MySQL persistence. If the deep learning branch is unavailable, the engine falls back to traditional strategies so that core learning flows remain usable. Chapters 1–3 present the introduction, key technologies, and requirements analysis.*

**Keywords**: intelligent recommendation; deep learning; forgetting curve; personalized learning; English vocabulary learning; dual-tower neural network

## 1 绪论

### 1.1 背景概述

#### 1.1.1 问题来源与研究动机

在全球化与高等教育普及背景下，**英语能力**仍是升学、科研与职业发展中的常见门槛，而**词汇量**是听、说、读、写各项技能的基础约束条件之一。传统课堂词汇教学受课时与班级规模限制，难以为每位学习者提供足量、及时的练习与反馈；**在线教育**与移动终端的普及，使「随时可学」成为可能，也对学习工具提出了更高要求：不仅要呈现词表，更要帮助学习者在有限时间内**提高留存率**与**迁移应用能力**。

从产业与产品形态看，网络与移动终端降低了在线学习的边际成本，慕课、音频视频课程与各类 **App** 并存，学习渠道从网站延伸到手机应用与内容社区。第三方行业报告曾指出，**中国在线教育市场规模**在近年已达到**千亿元量级**（具体年份与口径以所引用的行业报告为准）。在语言学习细分领域中，背单词类应用用户基数大，「百词斩」「墨墨背单词」「扇贝单词」等产品在应用商店中累计下载量较高，反映出**碎片化、高频次**的词汇学习需求。

以英语词汇记忆软件为例，不同产品在交互与策略上各具特点：有的侧重**图像联想**，有利于提升初学兴趣；有的强调**间隔重复与复习队列**，更贴近记忆规律，但可能带来「每日复习耗时较长」等与用户备考节奏之间的张力。用户需要在多款产品间比较与试错，**选型成本**会间接影响持续学习的意愿。与此同时，部分产品功能设置趋同，**差异化不足**时，**个性化学习目标**（如难度节奏、例句深度、复习强度）难以被充分满足。易观等机构曾发布**外语学习类 App 活跃度**等榜单类数据，可用于说明头部产品与细分赛道竞争格局。

从学习科学角度看，机械重复虽能在短期内提高识别率，却容易忽视**个体差异**与**记忆随时间衰减**的规律。心理学中的**遗忘曲线**与**间隔重复**思想表明：在接近遗忘的时点进行复习，往往比均匀重复更高效。另一方面，不同学习者的已有基础、学习目标与可用时间不同，**一刀切的词书顺序**难以兼顾「巩固旧词」与「拓展新词」的平衡。因此，将**学习行为数据化**（答题正确率、响应时间、复习次数等），并在此基础上做**难度与内容调度**，成为智能词汇类应用的核心技术路径之一。

近年来，**推荐系统**在电商、信息流等领域广泛应用；在教育场景中，推荐可用于在大量候选词中挑选**当前最应学习**的条目，并与**多题型练习**（选择、翻译、拼写等）、**闯关与评测**等形式结合，提高参与度与可观测的学习成果。学界对学习管理平台（LMS）与推荐算法在教育场景中的应用有持续综述与案例研究（如 Murad 等对相关文献的梳理）；国内也有基于协同过滤、神经网络等的**学习资源推荐**研究。深度学习技术的发展使得**用户—内容协同建模**（如双塔结构）在推荐任务中得以实践，但在毕业设计规模下，同样需要兼顾**可解释性、可部署性与失败降级**，避免「模型不可用时系统不可用」。

基于上述背景，本课题设计并实现 **SmartVocab（智能英语词汇学习推荐系统）**。系统采用 **B/S 架构**，以浏览器为客户端，服务端提供 **RESTful JSON API**；业务覆盖用户注册与登录、词库批量导入与导出、新词学习与复习会话、**多策略词汇推荐**、学习计划、**等级评测**、**闯关**与学习统计等。数据层采用 **MySQL**，将用户、词汇、学习记录、会话、评测与计划等实体组织为**十一张数据表**（逻辑模型与字段设计见**第 4 章**）；推荐层融合难度、词频、学习历史、随机探索等策略，并集成基于 **PyTorch** 的双塔深度学习推荐（**Python 依赖清单**中含 PyTorch；运行期若模型或组件初始化失败则**自动回退**传统算法）。系统还实现基于学习记录的**遗忘曲线**相关统计接口，为复习节奏与学习报告提供数据支撑。

本课题的**实践意义**在于：形成一套**需求—设计—实现一脉相承**的完整案例，便于在毕业论文中论述软件工程方法在垂直领域信息系统中的落地；**理论意义**在于：将推荐、记忆衰减与在线学习流程结合，对同类教学应用具有可复用的架构参考。

#### 1.1.2 研究边界与术语说明

为增强论述清晰度，本文对研究范围作如下**边界约定**：

- **范围以内**：Web 浏览器访问；服务端提供 JSON API；MySQL 持久化；JWT 鉴权；词汇学习、复习、推荐、计划、评测、闯关与统计等**本课题目标范围内**的能力。

### 1.2 发展状况

#### 1.2.1 国外发展概况

国外面向语言学习的商业产品与开源工具较多强调**间隔重复算法**（如基于 SM-2 等思路的调度）、**跨设备同步**与**用户社区**。代表性产品如 **Duolingo** 以游戏化路径与多语言支持见长，**Memrise** 侧重多媒体记忆与社区内容，**Quizlet** 以闪卡与协作学习为主；它们在**自适应学习、UGC 内容**等方面各有取舍：有的更偏入门与黏性，有的在词汇深度、内容品控或专业路径上存在局限。算法层面，闭源服务往往不公开特征与模型细节，但行业共识是：**用户建模 + 内容难度校准 + 复习调度**三者缺一不可。对高校教学与毕业设计而言，可完整阅读后端与数据表的开源实现仍相对有限。

#### 1.2.2 国内发展概况

国内互联网环境下，背单词类应用用户基数大，产品在**词书版权**、**打卡与社交**、**应试适配**（四六级、考研等）上竞争激烈。不少产品采用相对**标准化的学习流程**：选择词书与目标考试 → 设定完成周期与每日任务 → 以题库类练习区分生熟词 → 建立生词本并按计划复习。该模式便于运营与内容组织，但当**每日词难度主要由预设词书决定**、而非充分结合个体动态水平时，**个性化深度**可能受限；若缺乏对用户多维特征的持续建模，则难以称为「真正的个性化」。多数产品以客户端或小程序为主，**论文可复现的、前后端分离且接口文档化**的完整系统并不常见。对于计算机类专业毕业设计而言，若仅停留在界面原型或静态词表，难以充分体现**数据库设计、接口安全、推荐逻辑与工程化部署**等能力点。

#### 1.2.3 存在问题与本文切入点

对同类背单词与学习类应用的调研，可归纳若干**可改进空间**：（1）**推荐策略单一或融合不足**：仅依赖固定词书顺序或单一规则时，难以充分利用用户与内容的多维特征；（2）**冷启动与数据稀疏**：新用户历史不足、用户—词交互稀疏时，推荐稳定性与多样性面临挑战；（3）**复习可计算性**：若系统不持久化学习记录与复习时间，则难以支撑遗忘曲线与统计分析；（4）**工程可验证性**：缺少可本地部署、可自动化自检的后端，不利于论文中的测试章节撰写。

本课题在**可控实现**前提下，将 **Flask + MySQL + JWT** 的工程化基线与 **多算法融合推荐 + 基于 PyTorch 的双塔深度学习推荐** 结合，并落地**学习计划、评测、闯关、统计**等模块，与「仅展示词表」类应用在功能与数据闭环上形成明确区分；全文各章对系统功能与数据的表述力求前后一致、客观。

#### 1.2.4 教育信息化与可复现交付

从工程教育视角看，**可复现**（能在另一台机器按文档跑通）已成为评价毕业设计等实践环节质量的重要维度。国家与高校层面持续推进**教育信息化**与**新工科**实践，鼓励学生将**数据库、Web 工程、算法应用**贯通到完整项目中。相较于仅提交截图或演示视频，本系统配套**数据库脚本、环境变量说明、健康检查与自动化自检**，使系统可被核查，从而提升论述**可信度**——**不作**无法核验的量化宣传（如「提升百分之多少」需真实对照实验支撑，本课题不作此类断言）。

### 1.3 主要研究内容

#### 1.3.1 研究目标

本课题目标可从**能力维度**概括如下（具体功能以第 3 章与实现为准）：

1. **个性化推荐**：构建多策略融合推荐引擎，集成基于 **PyTorch** 的双塔深度学习分支与难度、词频、学习历史、随机探索等策略；特征体系上体现 **20** 维单词特征与 **15** 维用户特征的建模思路。

## 2 关键技术分析

选择合适的技术栈是系统可维护、可部署的前提。本项目以 **Python 生态**实现业务与算法，以 **MySQL** 保证事务与关联数据一致性，以 **JWT** 支撑无状态扩展；推荐层采用 **PyTorch** 实现双塔深度学习，**Python 依赖清单**已声明 PyTorch，安装依赖后即具备运行条件。若运行期深度学习推荐器初始化失败或缺少可用模型，推荐引擎**回退**至难度、词频、学习历史等传统策略，主流程不受影响。以下分节说明理由与要点。

### 2.1 系统总体架构

SmartVocab 采用 **B/S 架构**与**前后端分离**：**表示层**为浏览器中的 HTML/CSS/JS；**应用层**为 Flask 提供的 API 服务；**数据层**为 MySQL。浏览器通过 **HTTP/HTTPS** 获取静态资源与 JSON 数据，不在客户端持久化业务规则，便于统一鉴权与审计。

前端为**单入口 Web 应用**：入口页面以 **ES Module** 方式加载脚本，在单页内组织多视图（登录、学习、统计等），减少构建工具依赖。后端通过**应用启动器**统一注册认证、词汇、学习、推荐、计划、评测、闯关、健康检查等**蓝图模块**，并配置跨域、请求体大小上限、JSON UTF-8 等。该划分有利于**按模块分工开发**与**按路由排查问题**，契合中等规模信息系统的设计习惯。

**表 2-1 系统逻辑分层与职责**

| 层次 | 职责 |
|------|------|
| 接口层 | 对外 REST 路由、参数校验、鉴权装饰、统一响应格式 |
| 领域逻辑层 | 词汇学习、评测组卷与判分、推荐引擎、遗忘曲线计算等 |
| 数据访问层 | 连接池、各业务表的增删改查与事务边界 |
| 表示层 | 静态页面、样式、前端请求封装与 Token 注入 |

### 2.2 Flask 与 RESTful 后端

**Flask** 属于轻量级 Web 框架，核心简单、扩展灵活，适合毕业设计周期内的迭代。与「单文件巨型路由」相比，**Blueprint** 将 URL 前缀与视图函数分组（如认证、词汇、学习等前缀），使代码与业务边界对齐。

接口层采用 **REST 风格**的资源化路径与 HTTP 动词组合（如计划的创建、查询、更新、删除），便于前端封装与自动化测试脚本复用。需注意：部分学习会话类操作在语义上更接近 **RPC**（如开始学习会话），论文中可表述为「**REST 为主、关键动作用 POST 表达**」，与业界常见实践一致。

受保护资源统一经**认证中间件**校验：从 **Authorization: Bearer** 解析 JWT，并与路径或请求体中的用户标识做一致性校验，降低水平越权风险。

### 2.3 前端技术

前端采用 **HTML5** 语义化结构、**CSS3** 布局与样式、**ECMAScript 模块化**。**API 请求封装模块**集中处理基地址、JSON 头与 Token 注入，避免每个视图重复拼接请求逻辑。相较于引入完整 SPA 框架，本方案**构建链路短**、**部署与演示成本低**，代价是大型状态管理与组件复用需更多自律——对本课题规模而言可接受。

### 2.4 MySQL 数据库

**MySQL** 为成熟开源关系型数据库，支持事务、索引与外键，适合存储用户、词汇、学习记录、会话、评测、计划等存在明确关联的数据。系统库采用 **utf8mb4** 字符集，以兼容多语言释义与符号。连接由**数据库连接管理**统一维护连接池，减轻高并发短请求场景下的连接开销。

数据库设计遵循第三范式为主、在查询热点上建索引为辅的策略；各数据表定义见**第 4 章**；结构变更可通过迁移机制在开发与部署环境间保持一致。

### 2.5 JWT 与系统安全基线

**JWT** 将用户信息以签名令牌形式下发，服务端无需集中会话存储即可验证请求，利于水平扩展。本系统在登录、注册成功后签发 Token，业务请求携带 **Authorization: Bearer**。密钥与过期时间由**认证模块**与环境变量配置，生产环境对**过短或占位密钥**进行拦截。

密码存储采用 **bcrypt** 哈希，避免明文或可逆加密。Flask 应用密钥用于会话与签名相关默认行为；**错误详情是否返回前端**在生产环境默认收敛，避免将堆栈泄露给客户端。**CORS** 通过白名单配置允许的来源；生产环境应避免长期滥用通配。可在反向代理侧配合启用 **HSTS** 等响应头，实现全站 HTTPS。

### 2.6 智能推荐与遗忘曲线

#### 2.6.1 多策略推荐引擎

推荐引擎中定义多类子策略并按权重融合，默认权重如表 2-2 所示（若后续调参以实现为准）。深度学习、难度匹配、词频、学习历史、随机探索共同构成推荐结果。

**表 2-2 推荐子策略默认权重**

| 策略含义 | 默认权重 |
|----------|:--------:|
| 基于难度 | 0.25 |
| 基于词频 | 0.20 |
| 基于学习历史 | 0.20 |
| 深度学习推荐 | 0.25 |
| 随机探索 | 0.10 |

**推荐业务配置**中还约束最大推荐条数、最低分数阈值等，用于过滤弱相关结果，见表 2-3。

**表 2-3 推荐与学习相关配置常量（节选）**

| 配置含义 | 典型取值 |
|----------|----------|
| 单次推荐结果上限 | 50 |

深度学习部分由**深度学习推荐器**实现；当组件初始化失败时，引擎**回退**到传统策略，保证「无模型也能学」。

#### 2.6.2 深度学习推荐器结构要点

实现中约定单词侧特征维度 **20**、用户侧 **15**，与特征向量长度一致；若历史特征长度不符，对向量做截断或零填充以对齐维度。该设计便于在「系统实现」章中配图说明**双塔交互**。个性化模型可按用户维度加载（具体存储路径与命名以实现为准）。

**遗忘曲线相关配置**给出基础间隔、最大间隔与遗忘率等参数（见表 2-4），具体计算由**遗忘曲线管理逻辑**与学习记录表字段（如下次复习时间、掌握程度等）协同完成。统计页通过 **GET /api/learning/forgetting-curve/用户编号** 获取可视化所需数据。艾宾浩斯遗忘曲线与间隔重复理论可作为复习调度与统计展示的理论支撑。

**表 2-4 遗忘曲线配置常量**

| 含义 | 取值 |
|------|------|
| 基础复习间隔（天） | 1 |
| 最大复习间隔（天） | 30 |
| 遗忘率参数 | 0.5 |

### 2.7 部署、测试与运维

开发环境可通过**开发启动入口**运行服务；生产环境推荐使用 **Gunicorn** 多进程承载，前置 Nginx 终止 TLS 与反向代理。系统提供容器镜像与编排示例及**环境变量模板**，便于部署。**健康检查接口**分别检测进程存活与数据库连通性；**冒烟自检脚本**用于部署或验收前的快速验证；自动化测试用例覆盖健康与认证等基础路径。

### 2.8 本章小结

本章在 B/S 与前后端分离前提下说明了分层架构与蓝图划分；阐述了 Flask/REST、原生模块化前端、MySQL（utf8mb4 与连接池）、JWT 与安全基线；归纳了多策略推荐权重、双塔特征维度及遗忘曲线配置；并简述 Gunicorn、健康检查与冒烟测试等部署与验证手段。上述内容为**第 3 章需求分析**划定技术边界，为**第 4 章系统分析与设计**提供术语与依据。

## 3 需求分析

### 3.1 概述

**系统参与者**：**用户（学习者）**为系统的主要使用者，涵盖不同基础与目标的英语词汇学习需求，通过浏览器访问学习、统计、计划等功能；**系统**指服务端软件及其数据存储，负责认证、词汇与学习会话管理、推荐计算、评测与闯关逻辑、遗忘曲线与统计接口等。需求描述中「用户—系统」交互与后文 REST API 划分相对应。

#### 3.1.1 建设目标

SmartVocab 面向**自主英语词汇学习**场景，建设目标包括：（1）为学习者提供**可登录的个性化学习空间**；（2）将**学习、复习、评测、计划**纳入同一数据闭环；（3）为教师或管理员（若配置管理员名单环境变量）提供**词库批量维护**能力；（4）为成果验收与文档撰写提供**可运行、可演示**的实现载体。

#### 3.1.2 用户角色与权限概要

系统涉及两类典型角色，如表 3-1 所示。实际部署中，**管理员名单**由环境变量控制词库导入导出权限：**未配置**时，当前实现允许**所有已登录用户**使用导入导出（便于演示）；**配置后**仅名单内用户具备该能力。

**表 3-1 用户角色与职责**

| 角色 | 主要职责 | 说明 |
|------|----------|------|
| 学员（注册用户） | 学习、复习、评测、闯关、查看统计与推荐、管理个人学习计划 | 经 JWT 认证后访问业务 API |
| 管理员（视配置而定） | 词库批量导入、导出 | 由环境变量指定用户名；未配置时行为见上文 |

#### 3.1.3 约束条件

系统依赖 **MySQL** 与 Python 运行环境；客户端需支持 **ES Module**。**PyTorch** 已列入 **Python 依赖清单**，与系统一并安装。若深度学习推荐子模块初始化失败或缺少可用模型文件，推荐引擎在算法路径上**降级**为传统策略，**不得**影响注册、学习、评测等主流程。本章需求与**第 4 章系统分析与设计**相衔接；账户安全增强、词库细粒度管理等可作为**后续优化方向**在结论与展望中讨论。

#### 3.1.4 可行性分析

从毕业设计周期与资源条件出发，对建设 SmartVocab 作如下可行性判断（**不引入未经验证的量化收益承诺**）：

- **技术可行性**：后端采用 Flask、MySQL、JWT 等成熟技术，与本科阶段 Python、数据库、Web 基础衔接；推荐与遗忘曲线逻辑以**可配置常量 + 可替换算法模块**实现，便于分模块说明。**PyTorch 随依赖清单安装**；若深度学习分支不可用，推荐引擎走**传统策略**（属运行期降级）。
- **经济可行性**：开发以开源软件与本地或虚拟机部署为主，无强制商业授权；硬件为普通 PC 即可。
- **操作可行性**：学习者通过浏览器访问；管理员通过环境变量配置跨域、数据库连接、管理员名单等即可部署；运维侧可通过健康检查接口判断服务与库是否可用。
- **社会与环境、法律可行性**：系统服务于个人学习场景，应注意个人信息与学习数据的合法收集与存储，遵守网络安全与个人信息保护相关规范。

#### 3.1.5 业务数据对象与闭环（概念层）

系统核心数据对象包括：**用户**、**词汇**、**用户学习记录**、**学习会话**、**推荐记录**、**评测试卷与结果**、**闯关关卡与进度**、**学习计划**等，其逻辑关系在**第 4 章**以数据表形式给出。业务闭环可概括为：用户登录后产生的答题与会话数据写入学习记录与会话 → 推荐引擎读取历史与词表特征 → 遗忘曲线与统计接口聚合展示 → 评测与计划模块进一步影响后续学习难度与数量。该闭环依赖**持久化**，而非仅前端临时状态。本章侧重业务目标与能力边界；数据库结构与接口细节见**第 4 章**，实现与界面见**第 5 章**。

从业务流程角度，学员典型路径为：**注册/登录 → 查看首页推荐或制定计划 → 新词学习或复习会话 → 参加等级评测或闯关 → 在统计页查看进度与遗忘曲线 → 周期性调整计划**。系统应在数据层沉淀**学习记录与会话**，以支撑推荐与统计的**可重复计算**。

### 3.3 详细功能需求

#### 3.3.1 基本信息管理

- **账户**：注册、登录；个人信息查询与更新接口；密码 **bcrypt** 存储；业务接口需 **JWT**。
- **词库**：批量导入、导出接口；权限规则见 **3.1.2**。
- **词汇元数据**：支持难度、CEFR、词库体系、词频等字段，支撑选题与推荐。

#### 3.3.2 评测模块

- **组卷与答题**：开始等级测试接口，支持选择、翻译、拼写等题型。
- **提交与评估**：提交答卷接口，自动计分并给出水平信息。
- **历史**：按用户查询评测历史。

#### 3.3.3 学习计划模块

- **CRUD**：创建、按用户查询、查询当前生效计划、按计划标识更新与删除。
- **与复习的关系**：复习节奏主要依托学习记录中的复习时间与复习会话协同，与「仅由计划引擎独立驱动」的形态相区别。

#### 3.3.4 单词学习模块

- **会话**：开始学习、复习会话、提交答案等完成学习与复习流程。
- **闯关**：关卡列表、进度、解锁、开始关卡、完成关卡等支撑关卡体验。

#### 3.3.5 智能复习模块

- **进度与统计**：学习进度、统计、历史记录等接口。
- **推荐列表**：按用户获取推荐列表接口。

#### 3.3.6 遗忘曲线与智能测试

- **数据接口**：遗忘曲线数据接口，供统计页图表使用。

#### 3.3.7 系统集成

- 统一错误处理、安全响应头、健康检查；支持自动化冒烟自检。
